Code Log 1: HTML Semantic Tags

1. Difference Non-semantic tags vs Semantic tags

Non-semantic tags, like 'div' and "span", do not describe the meaning of the content inside them, they are mainly used for styling or grouping. Semantic tags, such as "header", "article", "section", and "footer", describe the role or meaning of the content, making it clearer for both developers and browsers.

Semantic tags - like "header"", "nav", "article", "footer": Describe what the content is (a header, navigation, article, etc.).

2. Benefits of Semantic Tags

Because screen readers and assistive devices can better comprehend the site structure thanks to semantic tags, accessibility is improved. Because they make it easier for search engines to recognize and rank valuable material, they help improve SEO. They also facilitate the reading and maintenance of code.

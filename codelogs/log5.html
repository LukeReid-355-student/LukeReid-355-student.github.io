Images

HTML vs CSS Image Presentation

Using an image in HTML with the <img> tag directly embeds the image into the page’s content. This is best for photos or visuals that are part of the page’s information. For example:

<img src="photo.jpg" alt="Example Image">

In contrast, using background-image in CSS adds the image as part of the design or layout, not the content itself. This method is useful for decorative images or backgrounds.

Functionally, <img> affects document structure and can be resized or given alt text for accessibility, while background-image is purely visual and controlled by CSS properties.

Other CSS Properties Affecting Background Images

  • background-size — Controls how the image fits its container (e.g., cover or contain).
  • background-repeat — Determines if the image repeats horizontally or vertically.
  • background-position — Defines where the image is placed inside the element (e.g., center, top right).

Dimensions vs Resolution

Dimensions refer to the physical width and height of an image (in pixels), while resolution describes the image’s pixel density or detail (often in DPI or PPI).

Extra Credit: Image File Formats

  • GIF — Supports animation and transparency; limited to 256 colors.
  • PNG — High-quality, lossless compression with transparency support.
  • JPG/JPEG — Compressed format for photos; small file sizes but some quality loss.
  • WebP — Modern format offering smaller sizes and good quality for web use.
  • SVG — Vector-based; scalable without losing quality, ideal for logos and icons.

Each format serves different purposes: PNGs for crisp graphics, JPEGs for photos, GIFs for simple animations, and SVGs for infinitely scalable vector art.

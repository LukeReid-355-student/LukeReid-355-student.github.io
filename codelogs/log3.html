CSS Units of Measurement

Absolute vs. Relative Units Explained

Absolute Units

Absolute units are constant regardless of screen size or other variables. They reflect precise measures (such as inches, points, or pixels) and will appear the same in every situation.

px (pixels)

The most commonly used absolute unit in web design. A pixel represents one dot on the screen.

pt (points)

Traditionally used in print design, one point equals 1/72 of an inch. Less common in web design today.

Relative Units

Relative units change based on the viewport, font size, or parent element. They are more adaptable and often superior for responsive design.

em

A relative unit determined by the parent element's font size. For example, if the parent font size is 16px, then 2em equals 32px.

rem

Similar to em, but it references the root element (<html>). This makes it more predictable, since it isn't affected by nested parent sizes.

Developers often prefer rem for consistent scaling across a site.

Example

html { font-size: 16px; }
.element { font-size: 1.5rem; } → 24px

Viewport Units

vw (viewport width)

Relative to the width of the browser window. For instance, 50vw equals 50% of the viewport's width.

vh (viewport height)

Relative to the height of the browser window. For example, 100vh makes an element stretch to the full height of the screen.
